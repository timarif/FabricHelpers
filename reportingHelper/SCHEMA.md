# Reporting Lakehouse — Canonical Schema

This document is the authoritative reference for the six Delta tables that
form the shared reporting lakehouse landing zone.  Every helper that writes
to this zone must produce rows that match these definitions exactly.

All timestamps are **UTC**.  All identifier columns are **strings** (GUIDs,
UPNs, or arbitrary run IDs).

---

## Table index

| Table | Grain | Primary key | Populated by |
|---|---|---|---|
| [`fact_scan_findings`](#fact_scan_findings) | One row per scanner finding | `finding_id` | `scannerHelper` |
| [`fact_downloads`](#fact_downloads) | One row per downloaded artifact | `download_id` | `downloaderHelper` |
| [`fact_mpe_lifecycle`](#fact_mpe_lifecycle) | One row per MPE lifecycle event | `event_id` | `mpeHelper` |
| [`dim_workspace`](#dim_workspace) | One row per Fabric workspace | `workspace_id` | Any helper |
| [`dim_item_type`](#dim_item_type) | One row per Fabric item type | `item_type` | Any helper |
| [`dim_severity`](#dim_severity) | One row per severity level | `severity` | `fabric-reporting` (seed) |

---

## fact_scan_findings

One row per finding produced by `scannerHelper`.

| Column | Type | Nullable | Notes |
|---|---|---|---|
| `finding_id` | STRING | **NOT NULL** | Primary key. UUID4 generated by the scanner. |
| `scan_run_id` | STRING | YES | Identifies the scanner run (e.g. ISO-8601 timestamp). |
| `workspace_id` | STRING | YES | Fabric workspace GUID. FK → `dim_workspace.workspace_id`. |
| `source_dated_partition` | STRING | YES | `ws_dated` partition label (e.g. `2024-01-15`). |
| `notebook_id` | STRING | YES | Fabric notebook item GUID. |
| `severity` | STRING | YES | `low` \| `medium` \| `high` \| `critical`. FK → `dim_severity.severity`. |
| `kind` | STRING | YES | Finding type / category (e.g. `url`, `entropy`, `potential_write`). |
| `redacted_snippet` | STRING | YES | Short code snippet with sensitive values redacted. |
| `url` | STRING | YES | URL found in the notebook, if applicable. |
| `detected_at` | TIMESTAMP | YES | UTC timestamp when the finding was detected. |

---

## fact_downloads

One row per artifact downloaded by `downloaderHelper`.

| Column | Type | Nullable | Notes |
|---|---|---|---|
| `download_id` | STRING | **NOT NULL** | Primary key. UUID4 generated by the downloader. |
| `run_id` | STRING | YES | Downloader run identifier. |
| `workspace_id` | STRING | YES | Fabric workspace GUID. FK → `dim_workspace.workspace_id`. |
| `item_id` | STRING | YES | Fabric item GUID. |
| `item_type` | STRING | YES | Fabric item type (e.g. `Notebook`). FK → `dim_item_type.item_type`. |
| `format` | STRING | YES | Download format (e.g. `ipynb`, `json`). |
| `downloaded_at` | TIMESTAMP | YES | UTC timestamp of the download. |
| `bytes` | INTEGER | YES | Artifact size in bytes (≥ 0). |
| `sha256` | STRING | YES | SHA-256 hex digest of the downloaded content. |

---

## fact_mpe_lifecycle

One row per managed-private-endpoint lifecycle event produced by `mpeHelper`.

| Column | Type | Nullable | Notes |
|---|---|---|---|
| `event_id` | STRING | **NOT NULL** | Primary key. UUID4 generated by mpeHelper. |
| `workspace_id` | STRING | YES | Fabric workspace GUID. FK → `dim_workspace.workspace_id`. |
| `mpe_id` | STRING | YES | Managed Private Endpoint GUID. |
| `target_resource_id` | STRING | YES | Azure resource ID of the MPE target. |
| `action` | STRING | YES | `created` \| `deleted` \| `approved`. |
| `actor` | STRING | YES | Identity (UPN or object ID) that performed the action. |
| `timestamp` | TIMESTAMP | YES | UTC timestamp of the lifecycle event. |

---

## dim_workspace

Workspace dimension — one row per Fabric workspace seen across all helpers.

| Column | Type | Nullable | Notes |
|---|---|---|---|
| `workspace_id` | STRING | **NOT NULL** | Primary key. Fabric workspace GUID. |
| `workspace_name` | STRING | YES | Human-readable display name. |
| `capacity_id` | STRING | YES | Fabric capacity GUID the workspace is assigned to. |
| `last_seen` | TIMESTAMP | YES | UTC timestamp when this workspace was last observed. |

---

## dim_item_type

Item-type dimension — one row per Fabric item type.

| Column | Type | Nullable | Notes |
|---|---|---|---|
| `item_type` | STRING | **NOT NULL** | Primary key (e.g. `Notebook`, `Pipeline`, `Dataflow`). |
| `family` | STRING | YES | Logical family (e.g. `data_engineering`, `data_factory`). |

---

## dim_severity

Severity dimension — seed data, one row per severity level.

| Column | Type | Nullable | Notes |
|---|---|---|---|
| `severity` | STRING | **NOT NULL** | Primary key: `low`, `medium`, `high`, `critical`. |
| `ordinal` | INTEGER | **NOT NULL** | Sort order: 0 = critical, 1 = high, 2 = medium, 3 = low. |
| `color` | STRING | YES | Suggested hex color (e.g. `#FF0000` for critical). |

Seed data:

| severity | ordinal | color |
|---|---|---|
| critical | 0 | `#C00000` |
| high | 1 | `#FF4500` |
| medium | 2 | `#FFA500` |
| low | 3 | `#FFD700` |
